Detailed Description

Part of resource schedule.

Run is the part of resource schedule consisting of:

  1. Performing zero or more start operations in the depot (at the start stop)
  2. Leaving the depot
  3. Performing one or more work operations at customers locations
  4. Returning to the depot
  5. Performing zero or more end operations in the depot (at the end stop)

Single resource can have multiple runs per schedule, e.g. if it's capacity isn't enough to deliver all orders in a single run. Different runs inside the single schedule may use different vehicles if there is such business requirement and the selected algorithms support such vehicles assignment.

List of stops in the run.

Run stops list is a subsequence of schedule stops list In fact only the schedule stores the list of stops. Runs just point to the range inside this list.

using Scheduler::Run::WorkStopIterator = WorkStopsList::iterator

Iterator to the work stop in the run.

using Scheduler::Run::WorkStopsList = RangeTypeAdapter<WorkStopsRange, WorkStop>

List of work stops in the run.

Like run stops list is a range inside the schedule stops list, run work stops list is a range inside run stops list. Being nested Range type it points to the range inside the schedule stops list and implicitly casts abstract Stop type to the WorkStop.

Run::WorkStopsList::iterator Scheduler::Run::unallocateOrder ( ConstWorkStopIterator  pos)

Unallocates order associated with the work stop at the specified position.

The following steps take place in this method:

  1. If order contains start operation, this operation is removed from the run's start stop.
  2. Work stop is detached from the run and destroyed
  3. If order contains end operation, this operation is removed from the run's end stop.
